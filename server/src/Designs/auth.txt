
--------------------  🔐 JWT ( JSON Web Token )  --------------------

🧩 What does a JWT look like?

- It’s just a long string like this:
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
- It has 3 parts:
HEADER.PAYLOAD.SIGNATURE
You don’t need to memorize just know:
Header → type of token
Payload → user data (like user_id)
Signature → security part (prevents cheating)

--------------------  📦 What is inside a JWT?  --------------------

Example payload (JSON):
{
  "userId": 5,
  "email": "test@gmail.com",
  "iat": 1710000000
}
👉 This is NOT encrypted (important!) It’s just encoded.
So, anyone can read it, but no one can change it

------------  🔁 How JWT works (SUPER SIMPLE FLOW)  ------------

1. Login
User sends (email + password) to POST /login

2. Server checks user
If correct:
👉 Server creates token, JWT(user data + secret key)

3. Server sends token to frontend. Frontend stores it in localStorage OR httpOnly cookie

4. Every request after login, frontend sends authorization: Bearer TOKEN_HERE

5. Backend verifies it
Server checks:
Is token valid?
Was it made by us?
Has it been changed?
If YES → allow access


--------------------  🛡️ Why JWT is useful?  --------------------

Without JWT:
You must log in every request ❌

With JWT:
Login once
Stay logged in ✔️
