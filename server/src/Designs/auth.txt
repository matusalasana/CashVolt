
------------------  🔐 JWT ( JSON Web Token )  --------------------

🧩 What does a JWT look like?
- It has 3 parts:
HEADER.PAYLOAD.SIGNATURE

Header → type of token
Payload → user data (like user_id)
Signature → security part (prevents cheating)


------------------  Server.js ftle  --------------------

app.use(cors({
  origin: "http://localhost:5173", // your frontend URL
  credentials: true
}));

origin → who can access your API
credentials: true → allow cookies
Without this: 👉 Cookies will NEVER be sent ❌

rate limiter
Prevents: 👉 Password guessing attacks


------------------  Login Controller  --------------------

const token = jwt.sign(...)   →  this line Creates identity proof
res.cookie("token", token, {...})  →  👉 Stores token in browser automatically in safe way (httpOnly: true, secure: false, sameSite: "lax"), you DON'T manually save it in React

Your current cookie:
JavaScript
secure: false
sameSite: "lax"
👉 This is fine for development
👉 NOT safe for production
✅ PRODUCTION VERSION
JavaScript
res.cookie("token", token, {
  httpOnly: true,
  secure: true,          // ✅ ONLY HTTPS
  sameSite: "none",      // ✅ allow cross-origin (frontend/backend separate)
  maxAge: 7 * 24 * 60 * 60 * 1000
});


------------------  Middleware (security guard)  --------------------

const token = req.cookies.token;  → 👉 Reads token automatically from browser cookie

const decoded = jwt.verify(token, SECRET) →  👉 Checks:
Is token real?
Is it expired?
Was it tampered with?

req.user = decoded; → 👉 Now EVERY route can access "req.user.userId"

Request → Middleware → Route → Response
